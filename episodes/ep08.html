Go에서 데이터 캡슐화의 기본 단위!

다른 언어에서는?

→ Java/C#: class

→ Python: dataclass

→ C: struct

Go 구조체의 특징

→ type + struct로 선언 ✅

→ 대문자 필드 = 외부 공개 (export) 🔓

→ 선언 시 제로값으로 자동 초기화 ⭐

→ 값 타입 + 메서드 부착 가능 💡

Go 1분 정리 EP.08

구조체 선언과 생성

// 구조체 정의 🏗️
type User struct {
    Name  string
    Age   int
    Email string
}

// 필드명 지정 ⭐ (권장!)
u := User{
    Name:  "상진수",
    Age:   28,
    Email: "jinsu@dev.com",
}
// 제로값 초기화: var u User 💡

💡 필드명 지정 + 제로값 활용이 Go의 정석!

Go 1분 정리 EP.08

필드 접근과 제로값

u := User{
    Name: "홍길동",
    Age:  32,
}
// 필드 접근: 점(.) 연산자
fmt.Println(u.Name)  // 홍길동
fmt.Println(u.Email) // "" 제로값!

// 필드 수정
u.Email = "sy@company.com"

// new()로 포인터 생성 ⭐
p := new(User) // *User, 제로값
p.Name = "상진수"

⭐ 미지정 필드는 제로값("", 0, false)!

Go 1분 정리 EP.08

메서드 정의

type Rect struct {
    W, H float64
}

// 값 리시버: 읽기만 📖
func (r Rect) Area() float64 {
    return r.W * r.H
}

// 포인터 리시버: 수정 가능 ✏️
func (r *Rect) Scale(f float64) {
    r.W *= f
    r.H *= f // 원본 수정!
}

💡 큰 구조체나 수정 필요 → 포인터 리시버!

Go 1분 정리 EP.08

구조체 임베딩

// 임베딩으로 코드 재사용 🧩
type BaseModel struct {
    ID        int
    CreatedAt string
}
type Product struct {
    BaseModel // 임베딩! ⭐
    Name  string
    Price int
}

p := Product{}
p.ID = 1        // 필드 승격!
p.Name = "노트북"
// p.BaseModel.ID와 동일 💡

💡 상속 대신 임베딩 — 필드와 메서드가 승격(promotion)!

Go 1분 정리 EP.08

생성자 패턴과 태그

// 생성자 패턴 ⭐ NewXxx
func NewUser(name string) *User {
    return &User{
        Name:  name,
        Email: name + "@dev.com",
    }
}
u := NewUser("상진수")

// 구조체 태그 🏷️
type Post struct {
    Title string `json:"title"`
    Body  string `json:"body,omitempty"`
    Views int    `json:"-"`
}

💡 omitempty: 빈값 생략, "-": JSON 제외!
