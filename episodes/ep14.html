패키지와 모듈이란? 🤔

패키지 = 관련 코드를 묶는 폴더 📂

모듈 = 패키지들의 묶음 + 버전 관리 📦

go.mod 파일이 모듈의 시작점! ⭐

대문자 = 공개, 소문자 = 비공개 🔒

다른 언어와 비교

→ Python: import 모듈 / pip install

→ Node.js: require/import / npm install

→ Java: package + Maven/Gradle

→ Go: package + go mod (내장!) ⚡

Go 패키지 핵심 특징

→ 폴더 = 패키지, 파일명 무관 ✅

→ go mod init으로 모듈 생성 🔧

→ go get으로 외부 패키지 설치 📥

→ 대소문자로 가시성 제어 (심플!) 🎯

Go 1분 정리 EP.14

패키지 선언과 import

// 모든 Go 파일은 패키지 선언 필수!
package main

import (
    "fmt"      // 표준 라이브러리
    "strings"  // 문자열 처리
    "time"     // 시간 관련
)

func main() {
    fmt.Println("Hello, Go!")
    fmt.Println(strings.ToUpper("go"))
    fmt.Println(time.Now())
}

⭐ import 그룹핑: 표준 / 외부 / 내부 순서!

Go 1분 정리 EP.14

가시성: 대문자 vs 소문자

package user

// 대문자 = 외부 공개 (exported) ✅
type User struct {
    Name  string // 공개
    Email string // 공개
    age   int    // 비공개!
}

// 공개 함수
func NewUser(name string) *User {
    return &User{Name: name, age: 0}
}

// 비공개 함수 (패키지 내부용)
func validate(email string) bool {
    return len(email) > 0
}

⭐ Go의 가시성 = 첫 글자 대소문자가 전부!

Go 1분 정리 EP.14

사용자 정의 패키지 만들기

// 프로젝트 구조
// myapp/
//   go.mod
//   main.go
//   calc/
//     math.go

// calc/math.go
package calc

func Add(a, b int) int { return a+b }
func Mul(a, b int) int { return a*b }

// main.go
package main
import "myapp/calc"
func main() {
    fmt.Println(calc.Add(3, 5)) // 8
}

💡 폴더명 = 패키지명으로 맞추는 게 관례!

Go 1분 정리 EP.14

init() 함수와 별칭 import

package config

var Settings map[string]string

// init: 패키지 로드 시 자동 실행!
func init() {
    Settings = map[string]string{
        "env": "production",
    }
    fmt.Println("설정 초기화 완료!")
}

// 별칭 import 활용
import (
    f "fmt"          // 별칭
    _ "image/png"    // 부작용만
    . "strings"      // 직접 접근
)
f.Println(ToUpper("go"))

⚠️ init()은 테스트 어렵게 만드니 최소한만!
