Go 1분 정리 EP.11

고루틴이란? 🤔

Go 런타임이 관리하는 경량 실행 단위

go 키워드 하나로 동시 실행 시작! ⭐

OS 스레드보다 훨씬 가볍고 빠름 (초기 ~2KB)

수만 개를 동시에 실행해도 OK! 🚀

OS 스레드 vs 고루틴

→ 스레드: ~1MB 스택, OS가 스케줄링

→ 고루틴: ~2KB 스택, Go 런타임 스케줄링

→ 스레드: 수천 개 → 성능 저하

→ 고루틴: 수만 개도 거뜬! 💪

고루틴 핵심 특징

→ go 키워드로 간단히 실행 ✅

→ M:N 스케줄링 (고루틴:OS스레드) ⚡

→ 스택 크기 자동 확장/축소 📏

→ 동기화: sync 패키지 + Channel 🔒

기본 고루틴 실행

func hello(name string) {
    fmt.Printf("안녕, %s!\n", name)
}

func main() {
    // go 키워드 하나로 동시 실행!
    go hello("Go")
    go hello("세계")

    // 익명 함수 고루틴
    go func() {
        fmt.Println("익명 고루틴!")
    }()
    time.Sleep(time.Second)
}

Go 1분 정리 EP.11

sync.WaitGroup으로 완료 대기

func worker(id int, wg *sync.WaitGroup) {
    defer wg.Done() // 완료 시 카운터 -1
    fmt.Printf("워커 %d 작업중\n", id)
    time.Sleep(time.Second)
    fmt.Printf("워커 %d 완료\n", id)
}
func main() {
    var wg sync.WaitGroup
    for i := 1; i <= 5; i++ {
        wg.Add(1) // 카운터 +1
        go worker(i, &wg)
    }
    wg.Wait() // 카운터 0이 될 때까지
    fmt.Println("전체 작업 완료!")
}

⭐ Add → go → Done → Wait 패턴이 기본!

Go 1분 정리 EP.11

sync.Mutex로 공유 자원 보호

var mu sync.Mutex
count := 0

var wg sync.WaitGroup
for i := 0; i < 5; i++ {
    wg.Add(1)
    go func() {
        defer wg.Done()
        mu.Lock()
        count++ // 안전한 접근
        mu.Unlock()
    }()
}
wg.Wait()
fmt.Println(count) // 정확히 5 ✅

⚠️ Mutex 없이 동시 수정 → 데이터 레이스!

Go 1분 정리 EP.11

실무 패턴: 워커 풀

// 워커 풀: N개 워커가 작업 소비 ⭐
jobs := make(chan int, 100)
var wg sync.WaitGroup
for w := 0; w < 3; w++ {
    wg.Add(1)
    go func() {
        defer wg.Done()
        for j := range jobs {
            process(j)
        }
    }()
}
// 작업 전송 후 채널 닫기
for j := 0; j < 20; j++ { jobs <- j }
close(jobs); wg.Wait()

💡 다음 슬라이드에서 errgroup 패턴!

Go 1분 정리 EP.11

errgroup: 고루틴 에러 관리

// errgroup: 고루틴 에러 관리 ⭐
// "golang.org/x/sync/errgroup"
g, ctx := errgroup.WithContext(ctx)
urls := []string{
    "https://api.a.com",
    "https://api.b.com",
}
for _, url := range urls {
    g.Go(func() error {
        return fetch(ctx, url)
    })
}
// 모든 완료 대기 + 첫 에러 반환
if err := g.Wait(); err != nil {
    log.Fatal(err) // 에러 전파!
}

⭐ errgroup = WaitGroup + 에러 전파 + 취소!
