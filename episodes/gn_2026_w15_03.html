왜 개발자가 주목해야 할까? 💡

📊 CLAUDE.md 100줄 이내가 최적 — LLM이 일관되게 따를 수 있는 지시는 150~200개 한계, 시스템 프롬프트가 이미 ~50개 선점해 실제 여유는 100~150개뿐

⚡ Hooks 하나로 파일 저장 즉시 Prettier 자동 실행 — CLAUDE.md 규칙 대신 hooks 쓰면 100% 결정론적 실행, LLM 지시 무시 불가

🤖 subagents는 독립된 컨텍스트 창 — 탐색·연구 작업을 메인 대화 노이즈 없이 분리, Explore 서브에이전트는 Haiku로 비용 최적화

🗂️ rules/에 paths: frontmatter 적용 → 백엔드 파일 열 때만 API 규칙 로드, 불필요한 컨텍스트 소비 제로

⚠️ 남이 만든 스킬 무분별 설치 금지 — 과도한 설정 누적 시 오히려 모델 응답 품질 저하, 간결함이 성능의 핵심

알고 쓰면 생산성 2배, 모르고 쌓으면 AI가 오히려 느려진다 🎯

실전 구조 완전 해부 🔍

🗂️ CLAUDE.md vs rules/ 분리 전략

CLAUDE.md는 전체 세션에 항상 로드 — 팀 공통 빌드/테스트 명령, 핵심 아키텍처 규칙만 유지. 200줄 넘어가면 rules/로 분리: `paths: ["**/*.test.ts"]` 같은 frontmatter로 특정 파일 열 때만 로드. 서브디렉토리도 자동 인식 (.claude/rules/frontend/react.md)

⚙️ settings.json Hooks 실전 패턴

PostToolUse + matcher: "Edit|Write" → 파일 수정 즉시 Prettier 자동 실행. Pre-hook으로 `rm -rf *` 같은 위험 명령 차단. 우선순위: CLI 플래그 > settings.local.json (gitignored) > settings.json (committed) > ~/.claude/settings.json

🤖 agents/ 커스텀 서브에이전트 설계

마크다운 + YAML frontmatter 파일 하나로 독립 에이전트 생성 — 자체 시스템 프롬프트, 도구 권한, 모델 선택, permission 모드 설정 가능. .claude/agents/(팀 공유, git commit) vs ~/.claude/agents/(개인 전용, 전 프로젝트에서 사용) 구분

🔌 .mcp.json + skills/ 활용

프로젝트 루트 .mcp.json — 팀 전체가 쓰는 GitHub·DB·API MCP 서버 공유, 비밀값은 `${GITHUB_TOKEN}` 환경변수 참조로 파일 노출 차단. skills/는 이제 commands/ 대체 — /skill-name 호출 또는 Claude 자동 실행, 서포팅 파일도 동일 폴더에 번들링 가능
