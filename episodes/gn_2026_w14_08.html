실전 적용 가이드 🔍

🗂️ CLAUDE.md: 100줄 이내로 유지

LLM이 일관되게 따를 수 있는 지시는 150~200개 한계. Claude 시스템 프롬프트가 이미 ~50개 사용 → 실제 여유는 100~150개뿐. Boris의 CLAUDE.md도 ~100줄. 매 줄 'Claude가 없으면 실수하나?' 자문. 반드시 지켜야 할 규칙은 Hooks로 — 100% 결정론적

⚙️ Plan Mode로 시작하기

Shift+Tab 2번 또는 --permission-mode plan으로 활성화. 읽기 전용 도구만 사용해 분석·계획만 수행. 7개 이상 파일 변경 계획은 컨텍스트가 가득 차 품질 저하 → 작은 태스크로 분할. Ctrl+G로 계획을 에디터에서 직접 편집

🔀 병렬 워크트리 실전

incident.io 팀: UI·빌드·테스트·백엔드 에이전트 4~5개 동시 실행 → JS 에디터 개선 작업 '2시간 → 10분' 달성. claude --worktree feature-name 한 명령으로 독립 체크아웃 생성. 실용적 상한선은 3~5개 — 더 많으면 컨텍스트 스위칭 비용이 역전

🧠 컨텍스트 윈도우 관리

컨텍스트 60% 이상이면 품질 저하 시작 (20~40%부터 저하). 코드 커밋 후 /clear로 컨텍스트 초기화가 핵심 습관. CLAUDE.md 내용도 매 세션 컨텍스트를 차지함 — 간결함이 성능
