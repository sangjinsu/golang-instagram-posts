Go 1분 정리 EP.09

포인터 선언과 역참조

val := 42
ptr := &val // &: 주소 획득
fmt.Println(ptr)  // 0xc0000b2008

// *: 역참조 (값 읽기)
fmt.Println(*ptr) // 42

// 역참조로 원본 값 변경 ⭐
*ptr = 100
fmt.Println(val)  // 100

// 다양한 포인터 선언 방식
var p1 *int       // nil
p2 := new(int)    // 제로값 할당
p3 := &val        // 기존 변수 참조

💡 &는 주소 획득, *는 역참조 + 값 수정

Go 1분 정리 EP.09

구조체 포인터와 자동 역참조

type Server struct {
    Host string
    Port int
    TLS  bool
}
// &T{}는 할당 후 포인터 반환 ⭐
s := &Server{
    Host: "0.0.0.0",
    Port: 8080,
}
// 자동 역참조: (*s).Host 불필요!
fmt.Println(s.Host) // 0.0.0.0
s.Port = 3000       // 직접 수정
s.TLS = true        // 원본 변경 💡

⭐ &T{} vs new(T): 결과는 동일한 *T

Go 1분 정리 EP.09

값 전달 vs 포인터 전달

// Go는 항상 pass by value!
func reset(n int) {
    n = 0 // 복사본만 변경
}
// 포인터 전달 → 원본 수정 가능
func resetPtr(n *int) {
    *n = 0 // 원본 변경 ⭐
}

score := 95
reset(score)
fmt.Println(score)    // 95
resetPtr(&score)
fmt.Println(score)    // 0
// 포인터 자체도 복사되어 전달됨 💡

⭐ Go는 항상 pass by value, 포인터도 복사!

Go 1분 정리 EP.09

nil 안전성과 방어 패턴

var p *int // 제로값은 nil
// *p → panic: nil dereference!

// nil guard 패턴 ✅
if p != nil {
    fmt.Println(*p)
}
// 생성자에서 nil 반환 활용 ⭐
func FindUser(id int) *User {
    if id <= 0 {
        return nil // 없음 표현
    }
    return &User{ID: id}
}
// if u := FindUser(1); u != nil

⚠️ nil 역참조 = panic! 항상 nil guard 필수

Go 1분 정리 EP.09

포인터 리시버와 참조 타입

// 포인터 리시버: 상태 변경 ✏️
func (s *Server) SetHost(h string) {
    s.Host = h // 원본 수정
}
// 값 리시버: 읽기 전용 📖
func (s Server) Addr() string {
    return s.Host + ":" + ...
}
// slice·map·chan은 내부 포인터 💡
func fill(s []int) {
    s[0] = 999 // 원본 변경됨
}
// 포인터 사용 기준 ⭐
// 큰 구조체, 상태 변경, nil 표현
// 공유 데이터, 인터페이스 구현

💡 값/포인터 리시버 혼용은 지양하세요!

Go 1분 정리 EP.09

📌 핵심 정리

✅ &로 주소 획득, *로 역참조 및 수정

✅ 포인터 산술 차단 → 메모리 안전 보장

✅ nil 제로값 + nil guard로 panic 방지

✅ 구조체 포인터 자동 역참조 (.접근)

✅ pass by value 원칙, 포인터로 원본 수정

💡 Go만의 꿀팁

💡 큰 구조체·상태 변경 → 포인터 리시버

💡 slice·map·chan은 포인터 전달 불필요

💡 &T{}와 new(T)는 동일한 *T 반환

💡 NewXxx() → *Xxx 생성자 패턴이 관례
