Go 1분 정리 EP.10

기본 인터페이스와 암시적 구현

type Speaker interface {
    Speak() string
}
type Dog struct{ Name string }

// implements 키워드 불필요! ⭐
func (d Dog) Speak() string {
    return d.Name + ": 멍멍!"
}
type Cat struct{ Name string }
func (c Cat) Speak() string {
    return c.Name + ": 야옹~"
}
var s Speaker = Dog{"바둑이"}
fmt.Println(s.Speak())

⭐ 메서드만 구현하면 자동으로 인터페이스 충족!

Go 1분 정리 EP.10

빈 인터페이스와 타입 단언

// 빈 인터페이스 = 모든 타입 수용
var x interface{}
x = 42
x = "hello"
x = []int{1, 2, 3}

// Go 1.18+ any 별칭 사용 ⭐
var y any = "world"

// 타입 단언 (Type Assertion)
str, ok := y.(string)
if ok {
    fmt.Println(str) // world
}
// ok=false면 제로값 반환 (안전!)

💡 any = interface{}, Go 1.18부터 사용 권장

Go 1분 정리 EP.10

인터페이스 합성 (Composition)

type Reader interface {
    Read(p []byte) (int, error)
}
type Writer interface {
    Write(p []byte) (int, error)
}
// 작은 인터페이스를 조합! ⭐
type ReadWriter interface {
    Reader
    Writer
}
// io 패키지의 실제 패턴 📖
// io.Reader → io.ReadWriter
// io.Closer → io.ReadCloser
// 작게 만들고, 필요하면 합성!

⭐ Go 표준 라이브러리의 핵심 설계 패턴!

Go 1분 정리 EP.10

타입 스위치와 다형성

func describe(i any) string {
    switch v := i.(type) {
    case int:
        return fmt.Sprintf("%d", v)
    case string:
        return fmt.Sprintf("%s", v)
    case Speaker:
        return v.Speak()
    default:
        return "알 수 없는 타입"
    }
}
fmt.Println(describe(42))
fmt.Println(describe("Go"))
fmt.Println(describe(Dog{"멍이"}))

💡 타입 스위치로 인터페이스의 실제 타입 분기!

Go 1분 정리 EP.10

실무 패턴: 인터페이스 설계 원칙

// Go 격언: 작게 만들어라 ⭐
// io.Reader → 메서드 1개
// fmt.Stringer → 메서드 1개
type Saver interface {
    Save(data []byte) error
}
// 인터페이스로 의존성 주입 💡
type App struct{ db Saver }
// 테스트: Mock 구현 주입
// 운영: 실제 DB 구현 주입
type MockDB struct{ called bool }
func (m *MockDB) Save(
    data []byte) error {
    m.called = true; return nil
}

Go 1분 정리 EP.10

컴파일 타임 체크와 의존성 주입

// Go 격언: 작게 만들어라 ⭐
// io.Reader → 메서드 1개
type Saver interface {
    Save(data []byte) error
}
type MockDB struct{ called bool }
func (m *MockDB) Save(
    data []byte) error {
    m.called = true; return nil
}
// 컴파일 타임 인터페이스 체크 ⭐
var _ Saver = (*MockDB)(nil)
// 인터페이스로 의존성 주입 💡
type App struct{ db Saver }
// 테스트: Mock, 운영: 실제 DB
