Go 1분 정리 EP.12

채널이란? 🤔

고루틴 간 데이터를 주고받는 파이프 🔗

make(chan type)으로 생성, <- 로 송수신! ⭐

보내고 받을 때까지 자동 블로킹 (동기화 내장)

타입 안전: chan int, chan string 등 📦

공유 메모리 vs 채널

→ 공유 메모리: Mutex 필요, 실수 위험

→ 채널: 메시지 전달, 자연스러운 동기화

→ Go 철학: 메모리 공유 ❌ 통신으로 공유 ✅

→ 채널 = 안전한 고루틴 대화 통로! 💬

채널 핵심 특징

→ make(chan T)로 간단히 생성 ✅

→ ch <- val 송신, val := <-ch 수신 ⚡

→ 방향 지정 가능 (송신/수신 전용) 🔒

→ close + range로 깔끔한 종료 패턴 🎯

Go 1분 정리 EP.12

기본 채널 사용

ch := make(chan string)

go func() {
    ch <- "안녕, 채널!" // 송신
}()

msg := <-ch // 수신 (블로킹 대기)
fmt.Println(msg)

// 여러 값 전달
nums := make(chan int)
go func() {
    nums <- 1
    nums <- 2
    nums <- 3
}()
fmt.Println(<-nums, <-nums, <-nums)

⭐ 수신자 없으면 송신 블로킹, 그 반대도 마찬가지!

Go 1분 정리 EP.12

버퍼드 채널

// 언버퍼드: 수신자 있어야 송신 가능
ch1 := make(chan int)

// 버퍼드: 버퍼 꽉 찰 때까지 논블로킹!
ch2 := make(chan int, 3)
ch2 <- 1  // 바로 진행 ✅
ch2 <- 2  // 바로 진행 ✅
ch2 <- 3  // 바로 진행 ✅
// ch2 <- 4 // 블로킹! 버퍼 꽉 참 😱

fmt.Println(len(ch2)) // 3 (현재 개수)
fmt.Println(cap(ch2)) // 3 (버퍼 크기)
fmt.Println(<-ch2)    // 1 (FIFO)

💡 버퍼 크기 = 비동기 처리 가능한 양!

Go 1분 정리 EP.12

채널 방향 지정

// 송신 전용: chan<- int
func producer(ch chan<- int) {
    for i := 0; i < 5; i++ {
        ch <- i
    }
    close(ch)
}
// 수신 전용: <-chan int
func consumer(ch <-chan int) {
    for v := range ch {
        fmt.Println(v)
    }
}
func main() {
    ch := make(chan int)
    go producer(ch)
    consumer(ch) // 0,1,2,3,4
}

⭐ 방향 지정 → 실수 방지 + 의도 명확!

Go 1분 정리 EP.12

range와 close

ch := make(chan int)

go func() {
    for i := 1; i <= 5; i++ {
        ch <- i
    }
    close(ch) // 더 이상 안 보냄!
}()

// range: 채널 닫힐 때까지 수신
for v := range ch {
    fmt.Println(v) // 1, 2, 3, 4, 5
}

// 닫힌 채널 확인
v, ok := <-ch
fmt.Println(v, ok) // 0 false

⚠️ 닫힌 채널에 송신하면 panic! 주의!

Go 1분 정리 EP.12

select 문

ch1 := make(chan string)
ch2 := make(chan string)

go func() { ch1 <- "하나" }()
go func() { ch2 <- "둘" }()

time.Sleep(time.Millisecond)

// 먼저 도착하는 채널 수신!
select {
case v := <-ch1:
    fmt.Println(v)
case v := <-ch2:
    fmt.Println(v)
case <-time.After(time.Second):
    fmt.Println("타임아웃!")
}

💡 select = 다중 채널의 switch문!

Go 1분 정리 EP.12

실전: Worker Pool 패턴

func worker(id int, jobs <-chan int,
    results chan<- int) {
    for j := range jobs {
        fmt.Printf("워커%d: 작업%d\n", id, j)
        results <- j * 2
    }
}
func main() {
    jobs := make(chan int, 100)
    results := make(chan int, 100)
    for w := 1; w <= 3; w++ {
        go worker(w, jobs, results)
    }
    for j := 1; j <= 9; j++ { jobs <- j }
    close(jobs)
    for i := 0; i < 9; i++ {
        fmt.Println(<-results)
    }
}

⭐ 워커 풀 = 채널 활용의 대표 실무 패턴!
