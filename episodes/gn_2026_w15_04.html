왜 개발자가 주목해야 할까? 💡

📊 4비트 TurboQuant — H100서 32비트 대비 어텐션 logit 연산 8배 빠름. KV 캐시는 LLM 추론 메모리의 60~80%를 차지하는 최대 병목, 6배 감소는 동일 GPU로 6배 더 긴 컨텍스트 처리 가능

🎯 캘리브레이션 데이터 0개 — 기존 GPTQ/AWQ는 모델별 캘리브레이션 셋 + 시간이 필요. TurboQuant는 random matrix 기반이라 어떤 파인튜닝 모델에도 즉시 drop-in, per-model 셋업 비용 제로

🧮 PolarQuant 작동 원리 — 좌표 쌍을 폴라 변환(반지름+각도)하고 random rotation 적용. 각도 분포가 예측 가능해져 블록당 정규화 상수 저장 비용이 사라짐

📈 LongBench 벤치마크 — 질의응답·코드 생성·요약 전 항목에서 KIVI 베이스라인 매칭 또는 초과, KV 메모리는 6배 적게 사용. Llama-3.1-8B서 0.9996 cosine similarity, 94% top-1 어텐션 매칭

💰 서빙 경제학 변화 — 발표 48시간 만에 SK하이닉스·삼성·마이크론 시총 1000억 달러 증발 (CNBC). 추론 비용 50%+ 절감, 70B 모델 컨텍스트가 109K → 536K로 확장 (RTX 5090 32GB 실측)

재학습 0초 · 캘리브레이션 0개 · 정확도 손실 0% — drop-in 한 줄로 6배 압축 🎯

실전 적용 완전 해부 🔍

🧮 2단계 알고리즘 PolarQuant + QJL

PolarQuant: d차원 벡터를 좌표 쌍씩 묶어 폴라 변환(반지름·각도) → 재귀적 변환으로 단일 radius + 각도 집합으로 압축. random rotation 후 각도 분포가 균일해져 정규화 상수 불필요. QJL: 남은 잔차에 Johnson-Lindenstrauss 변환을 적용해 1비트 sign(+1/-1)으로 환원, PolarQuant의 체계적 편향을 제거

🎚️ Asymmetric Bit Allocation (K4/V2)

PyTorch 구현체 검증 — Key는 4비트, Value는 2비트 할당이 sweet spot. 키는 어텐션 focus를 결정해 정밀도 민감, 값은 평균화로 오차가 자연 상쇄. K4/V2 = 5.1x 압축, 보호 레이어 적용 시 3.6x. 8K 컨텍스트서 cosine similarity 0.9996 달성

🚀 Drop-in 통합 — vLLM·TGI·llama.cpp

기존 서빙 프레임워크에 KV 캐시 압축 레이어 한 줄로 삽입. random matrix 기반이라 모델 가중치 무손상, 체크포인트 변경 0건. llama.cpp 디스커션(#20969)에서 통합 진행 중, Rust 크레이트 turboquant-rs도 공개. 파인튜닝된 Llama·Mistral·Gemma 그대로 사용

⚠️ QJL 잔차 보정의 함정

PyTorch 재현 시 6개 독립 팀이 확인한 사실 — QJL 잔차 단계가 어텐션 워크로드선 오히려 성능 저하. softmax가 분산을 지수적으로 증폭해 unbiased estimator가 역효과. 실전 배포 시 MSE-only 구성이 MSE+QJL보다 일관되게 우수, 논문 그대로 쓰지 말고 워크로드별 ablation 필수
