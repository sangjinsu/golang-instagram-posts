에러 처리란? 🤔

Go에는 try-catch가 없다! 🚫

error는 인터페이스, 에러도 '값'이다 ⭐

함수가 (결과, error)를 함께 반환! 📦

if err != nil 패턴으로 직접 처리 🔍

try-catch vs Go 에러

→ try-catch: 예외를 던지고 잡는 방식

→ Go: 에러를 값으로 반환하는 방식

→ Go 철학: 에러를 숨기지 말고 명시적으로! ✅

→ 에러 = 프로그램의 정상 흐름 일부! 💡

에러 처리 핵심 특징

→ error 인터페이스 = Error() string ✅

→ 다중 반환값으로 에러 전달 ⚡

→ errors.New(), fmt.Errorf() 생성 🔧

→ errors.Is(), errors.As() 비교 🎯

Go 1분 정리 EP.13

기본 에러 처리

func divide(a, b float64) (float64, error) {
    if b == 0 {
        return 0, errors.New("0으로 나눌 수 없음")
    }
    return a / b, nil // 성공 시 nil
}

func main() {
    result, err := divide(10, 0)
    if err != nil {
        fmt.Println("에러:", err)
        return
    }
    fmt.Println(result)
}

⭐ if err != nil = Go의 가장 유명한 패턴!

에러 생성 방법

// 1. errors.New() - 간단한 에러
err1 := errors.New("파일을 찾을 수 없음")

// 2. fmt.Errorf() - 포맷팅 에러
err2 := fmt.Errorf("포트 %d 사용 중", 8080)

// 3. 센티널 에러 - 미리 정의!
var (
    ErrNotFound = errors.New("not found")
    ErrTimeout  = errors.New("timeout")
)

// 센티널 에러 비교
if err == ErrNotFound {
    fmt.Println("찾을 수 없습니다")
}

💡 센티널 에러 = 패키지 수준 전역 에러 상수!

커스텀 에러 타입

type ApiError struct {
    Code    int
    Message string
}
// error 인터페이스 구현
func (e *ApiError) Error() string {
    return fmt.Sprintf(
        "[%d] %s", e.Code, e.Message,
    )
}
func fetchUser(id int) error {
    return &ApiError{
        Code: 404, Message: "유저 없음",
    }
}
// 출력: [404] 유저 없음

⭐ Error() string만 구현하면 에러 타입!

에러 래핑 (Go 1.13+)

// %w로 에러 래핑 (원본 보존!)
func readConfig() error {
    _, err := os.Open("config.yaml")
    if err != nil {
        return fmt.Errorf(
            "설정 로드 실패: %w", err,
        )
    }
    return nil
}
// errors.Is() - 에러 체인 검색
if errors.Is(err, os.ErrNotExist) {
    fmt.Println("파일이 없습니다")
}
// errors.As() - 타입 추출
var apiErr *ApiError
errors.As(err, &apiErr)

💡 %w 래핑 → Is/As로 체인 탐색 가능!

defer, panic, recover

func safeDiv(a, b int) (result int) {
    defer func() {
        if r := recover(); r != nil {
            fmt.Println("복구:", r)
            result = 0
        }
    }()
    return a / b // b=0이면 panic!
}

func main() {
    fmt.Println(safeDiv(10, 0)) // 0
    // panic("심각한 오류!") 직접 호출
    // recover()는 defer 안에서만!
}

⚠️ panic은 정말 복구 불가능할 때만 사용!
