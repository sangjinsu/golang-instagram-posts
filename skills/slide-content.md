# 슬라이드 콘텐츠 작성 스킬

## 개요

에피소드별 콘텐츠를 JSON 형식으로 작성하는 규칙을 정의한다.
JSON 파일은 `episodes/` 디렉토리에 저장하며, HTML 생성 스크립트의 입력으로 사용된다.

## JSON 스키마

```json
{
  "episode": 8,
  "title": "구조체",
  "title_en": "Struct",
  "difficulty": "★★",
  "hook": "나만의 데이터 타입을 만들자! 🏗️",
  "next_ep": {
    "number": 9,
    "title": "포인터"
  },
  "slides": [
    {
      "slide_number": 1,
      "type": "thumbnail",
      "content": {
        "series_name": "Go 기초 문법",
        "series_sub": "1분 정리 ⚡",
        "ep_label": "EP.08",
        "topic": "구조체 (Struct)",
        "hook": "나만의 데이터 타입을 만들자! 🏗️"
      }
    },
    {
      "slide_number": 2,
      "type": "concept",
      "content": {
        "question": "구조체란? 🤔",
        "explanation": [
          "여러 데이터를 하나로 묶는 타입",
          "이름표 붙은 서랍장과 같아요! 🗄️"
        ],
        "comparison": {
          "label": "다른 언어에서는?",
          "items": [
            "Java/C#: class",
            "Python: dataclass",
            "C: struct"
          ]
        },
        "features": {
          "label": "Go 구조체의 특징",
          "items": [
            "type + struct로 선언 ✅",
            "클래스가 없는 대신 구조체 사용! ⭐",
            "메서드를 붙일 수 있음 💡",
            "값 타입 (복사됨) 📋"
          ]
        }
      }
    },
    {
      "slide_number": 3,
      "type": "code",
      "content": {
        "title": "구조체 선언과 생성",
        "code": "// 구조체 정의\ntype User struct {\n    Name string\n    Age  int\n    Email string\n}\n\n// 생성 방법 1: 필드명 지정 ⭐\nu1 := User{\n    Name:  \"고퍼\",\n    Age:   25,\n    Email: \"go@dev.com\",\n}\n\n// 생성 방법 2: 순서대로\nu2 := User{\"러스트\", 30, \"rs@dev.com\"}",
        "note": "💡 필드명 지정 방식을 권장해요!"
      }
    }
  ]
}
```

## 슬라이드 타입별 작성 규칙

### thumbnail (1장)

| 필드 | 값 | 비고 |
|------|-----|------|
| `series_name` | `"Go 기초 문법"` | 고정값 |
| `series_sub` | `"1분 정리 ⚡"` | 고정값 |
| `ep_label` | `"EP.XX"` | EP 번호 형식 |
| `topic` | `"주제명 (영문명)"` | 한국어 + 영문 |
| `hook` | 한줄 훅 + 이모지 | 필수 |

### concept (2장)

| 필드 | 값 | 비고 |
|------|-----|------|
| `question` | `"XXX란? 🤔"` 형식 | 필수 |
| `explanation` | 2~3줄 쉬운 설명 | 비유 포함 |
| `comparison` | 다른 언어와 비교 | 선택 |
| `features` | Go만의 특징 3~4개 | 필수 |

### code (3~7장)

| 필드 | 값 | 비고 |
|------|-----|------|
| `title` | 슬라이드 소제목 | 한국어, 필수 |
| `code` | Go 코드 문자열 | 필수 |
| `note` | 하단 팁 메시지 | 선택 |

**코드 작성 제약:**
- 한 줄 최대 **40자**
- 최대 **15줄**
- 한국어 주석 포함
- 이모지 주석 활용 (⭐ 💡 ⚠️ 🔥)

### summary (8장)

| 필드 | 값 | 비고 |
|------|-----|------|
| `points` | 핵심 정리 항목 4~5개 | ✅ 이모지 접두사 |
| `tips` | Go만의 꿀팁 3~4개 | ✅ 이모지 접두사 |
| `next_preview` | 다음 편 예고 | 필수 |
| `cta` | `"💾 저장 \| ❤️ 좋아요 \| 👤 팔로우"` | 고정값 |

## Go 코드 작성 가이드라인

### 스타일

- **들여쓰기**: 탭 대신 스페이스 4칸 (이미지 가독성)
- **변수명**: 한국어 맥락에 맞는 예시 (`score`, `name`, `age` 등)
- **주석**: 반드시 한국어로 작성
- **출력 결과**: 주석으로 표시 (`// 출력: ...`)

### 코드 색상 매핑 (HTML 변환 시 적용)

| 대상 | CSS 클래스 | 색상 | 키워드 |
|------|-----------|------|--------|
| 키워드 | `.kw` | `#ff7b72` | `func`, `var`, `const`, `if`, `for`, `return`, `type`, `struct`, `range`, `switch`, `case`, `defer`, `go`, `chan`, `select`, `import`, `package`, `map`, `make`, `append`, `len`, `cap`, `delete`, `nil`, `true`, `false`, `break`, `continue` |
| 문자열 | `.str` | `#a5d6ff` | `"..."`, `` `...` `` |
| 숫자 | `.num` | `#79c0ff` | 정수, 실수 |
| 주석 | `.cm` | `#8b949e` | `// ...` |
| 타입 | `.tp` | `#7ee787` | `string`, `int`, `bool`, `float64`, `error`, `byte`, `rune` + 사용자 정의 타입 |
| 함수명 | `.fn` | `#d2a8ff` | 함수 선언 시 이름, `fmt.Println` 등 |
| 연산자 | `.op` | `#ffa657` | `:=`, `=`, `==`, `!=`, `<`, `>`, `+`, `-`, `*`, `/` |

### 슬라이드별 코드 난이도 배분

| 슬라이드 | 난이도 | 내용 |
|----------|--------|------|
| 3장 | 기본 | 선언, 생성 |
| 4장 | 핵심 | Go만의 특별한 문법 |
| 5장 | 활용 | 실전 패턴 |
| 6장 | 심화 | 주의사항, 비교 |
| 7장 | 고급 | 실무 팁, 복합 활용 |

## 톤앤매너

- 친근하고 캐주얼한 말투 (`"~해요"`, `"~입니다"`)
- 이모지를 자연스럽게 활용
- 어려운 개념은 비유로 먼저 설명
- **"Go만의 특징"** 을 매번 강조
- 다른 언어 경험자가 이해하기 쉽게 비교 제공
- 실무 활용성 언급 (`"실무에서는 이렇게!"`)

|----------|-----------|
